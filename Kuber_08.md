# Домашнее задание к занятию «Компоненты Kubernetes»

## Рассчитать требования к кластеру под проект.
### master-ноды
Для отказоустойчивого кластера Kubernetes нужно минимум 3 master-ноды (control plane) в первую очередь для БД etcd. Рекомендуемый минимум для master: 4 CPU, 8 ГБ RAM каждая.

### worker-ноды
#### База данных (отказоустойчивая, 3 копии)
  - 4 ГБ RAM  
  - 1 CPU Core
  - Итого 12 ГБ RAM
  - Итого 3 CPU Core
#### Кеш (отказоустойчивый, 3 копии)
  - 4 ГБ RAM  
  - 1 CPU Core
  - Итого 12 ГБ RAM
  - Итого 3 CPU Core 
#### Фронтенд (5 копий)
  - 0.05 ГБ RAM  
  - 0.2 CPU Core
  - Итого 0.25 ГБ RAM
  - Итого 1 CPU Core
#### Бекенд (10 копий)
  - 0.6 ГБ RAM  
  - 1 CPU Core
  - Итого 6 ГБ RAM
  - Итого 10 CPU Core
#### Итого по всем приложениям
  - Итого 30.25 ГБ RAM
  - Итого 17 CPU Core

#### Добавление служебных ресурсов на ноду
На каждой worker-ноде часть ресурсов забирают: kubelet, container runtime, kube-proxy и т.д.  
Рекомендация: оставлять не менее 15–20% от ресурсов ноды незанятыми под системные нужды.  
Возьмём 20% запаса:  
Нода: 8 CPU, 16 ГБ RAM  
Доступно для приложений: 8 × 0.8 = 6.4 ядра, 16 × 0.8 = 12.8 ГБ RAM  
Теперь посчитаем, сколько нод нужно:  
Суммарные ресурсы подов: 17 CPU, 30.25 ГБ RAM.  
По CPU: 17 / 6.4 ≈ 2.66 → минимум 3 ноды (без запаса на отказ).  
По RAM: 30.25 / 12.8 ≈ 2.36 → минимум 3 ноды (без запаса на отказ). 
Лимит по ресурсам — 3 ноды.  
Добавляем N+1 на отказ одной ноды: 3 + 1 = 4 ноды.  
Проверяем при отказе 1 ноды из 4: остаётся 3 ноды × 6.4 CPU = 19.2 CPU (требуется 17 — ок), 
3 × 12.8 RAM = 36.4 ГБ (требуется 30.25 — ок).  

#### Предлагается вариант с учетом отказоустойчивости:
- 4 worker nodes  
- 16 GB RAM  
- 8 CPU Cores  

## По результату - требования к кластеру под проект.
- 3 master nodes 8GB RAM, 4 CPUs  
- 4 worker nodes 16GB RAM, 8 CPUs
