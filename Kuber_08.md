## Рассчитать требования к кластеру под проект
### master-ноды
Для отказоустойчивого кластера Kubernetes нужно минимум 3 master-ноды (control plane). Рекомендуемый минимум для master: 4 CPU, 8 ГБ RAM каждая в первую очередь для БД etcd.

### worker-ноды
#### База данных (отказоустойчивая, 3 копии)
  - 4 GB RAM  
  - 1 CPU Core
  - Итого 12 GB RAM
  - Итого 3 CPU Core
#### Кеш (отказоустойчивый, 3 копии)
  - 4 GB RAM  
  - 1 CPU Core
  - Итого 12 GB RAM
  - Итого 3 CPU Core 
#### Фронтенд (5 копий)
  - 0.05 GB RAM  
  - 0.2 CPU Core
  - Итого 0.25 GB RAM
  - Итого 1 CPU Core
#### Бекенд (10 копий)
  - 0.6 GB RAM  
  - 1 CPU Core
  - Итого 6 GB RAM
  - Итого 10 CPU Core
#### Итого по всем приложениям
  - Итого 30.25 GB RAM
  - Итого 17 CPU Core

#### Добавление служебных ресурсов на ноду
На каждой worker-ноде часть ресурсов забирают: kubelet, container runtime, kube-proxy и т.д.  
Рекомендация: оставлять не менее 15–20% от ресурсов ноды незанятыми под системные нужды.  
Возьмём 20% запаса:  
Нода: 8 CPU, 16 ГБ RAM  
Доступно для приложений: 8 × 0.8 = 6.4 ядра, 16 × 0.8 = 12.8 ГБ RAM  
Теперь посчитаем, сколько нод нужно:  
Суммарные ресурсы подов: 17 CPU, 30.25 ГБ RAM.  
По CPU: 17 / 6.4 ≈ 2.66 → минимум 3 ноды (без запаса на отказ).  
По RAM: 30.25 / 12.8 ≈ 2.36 → минимум 3 ноды.  
Лимит по ресурсам — 3 ноды.  
Добавляем N+1 на отказ одной ноды: 3 + 1 = 4 ноды.  
Проверяем при отказе 1 ноды из 4: остаётся 3 ноды × 6.4 CPU = 19.2 CPU (требуется 17 — ок), 
3 × 12.8 RAM = 36.4 ГБ (требуется 30.25 — ок).

Для работы кластера Kubernetes рекомендуется, как минимум, 3 master nodes из соображений обеспечения непротиворечивости (quorum) данных хранилища etcd.  
Рекомендуемые ресурсы для master node:

- 8 GB RAM
- 2 CPU Cores
- не менее 100 GB хранилища с производительностью не ниже 400 IOPS

Из описания приложения следует, что:

- для размещения СУБД необходимо 3 независимых node, на каждом узле СУБД потребует:
  - 4 GB RAM
  - 1 CPU Core
- для размещения сервиса кэширования необходимо 3 независимых node, на каждом узле сервис потребует:
  - 4 GB RAM
  - 1 CPU Core
- для размещения реплик бэкенд можно использовать те же 3 node, на которых разворачиваются СУБД и кэш, это потребует на каждом из 3 узлов:
  - примерно 0,6 x 10 / 3 + 0,6 = 2,6 GB RAM
  - примерно 1 x 10 % 3 + 1 = 4 CPU Cores
  - хранилище для persistentVolume-s
- для размещения реплик фронтенд так же можно использовать имеющиеся 3 node, это потребует на каждом из worker узлов:
  - примерно 50 x 5 % 3 + 50 = 130 MB RAM
  - примерно 0,2 x 5 / 3 + 0,2 = 0,5 CPU Cores
- для организации облачного хранилища можно предусмотреть ресурсы для развертывания облачного хранилища типа CEPH, GlusterFS или minio на 3 nodes
  - от 8 GB RAM
  - от 4 CPU Cores

Для поддержания собственной работоспособности worker node рекомендуется для каждого узла:

- 4 GB RAM
- 2 CPU Cores
