# Домашнее задание к занятию 1.  «Введение в виртуализацию»
## Задача 2.
1. Высоконагруженная база данных MySql, критичная к отказу.
  
Должна располагаться на физических серверах, так как более полно будут использоваться ресурсы без побочных нагрузок, с кластеризацией на уровне OS - pacemaker + DRBD или MySQL Replication.

2. Различные web-приложения.
  
Используют как правило однородную ОС и должны раполагаться на виртуализации уровня ОС - docker.

3. Windows-системы для использования бухгалтерским отделом.

Это паравиртуализация XEN, так как для бухгалтерии требуется стабильная работы систем.

4. Системы, выполняющие высокопроизводительные расчёты на GPU.

Должны располагаться на множестве отдельных физических специализированных серверах для полного использования физических ресурсов. Эти сервера должны распараллеливать задачи.
## Задача 3.
1. 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.

Для такой задачи лучше всего подойдет Виртуализация VMware vSphere, которая хорошо работает с разными ОС на виртуальных машинах и обладает инструментами управления нагрузкой и репликации данных. Так же есть автоматичское создание снапшотов.

2. Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.

Здесь лучшим будет использование XEN, так как он обладает большей производительнотью, но подойдет и KVM.

3. Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.

Тут я бы выбрал VMWare ESXI или KVM как максимально совместимые для виртуализации Windows-инфраструктуры.

4. Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.

Для данной задачи максимально подходит docker, так как требуется тетсировать только приложение, то использование готовых контейнеров с разными OS очень выгодно.
   
## Задача 4.
Проблемы и недостатки гетерогенной среды виртуализации - в первую очередь это обслуживание, мониторинг и диагностика неисправностей. Так же сопровождение, в смысле подготовки сотрудников, "разного" обрудования. Еще есть серьезные вопросы по лицензированию. Необходимо выбрать универсальную систему виртуализации и по возможности переводить существующие приложения на нее. Если есть выбор то надо по возможности создавать единую систему виртуализации.
